# Issue: Documentation Website for MondayMoney

## Description
Create a dedicated documentation website for MondayMoney to provide users and developers with a centralized resource for information, guides, and technical details. The website will be served via GitHub Pages.

## Reference
The documentation style and structure should be inspired by [Sinkhole's documentation](https://thiagocolen.github.io/Sinkhole/), which uses MkDocs with the Material theme.

## Requirements
- Set up **MkDocs** with the **Material for MkDocs** theme.
- Configure a GitHub Action for automated deployment to GitHub Pages.
- Include the following sections:
    - **Features**: Overview of MondayMoney's core capabilities (Transaction tracking, Category management, Tagging, CSV Imports).
    - **Technical**: Architecture overview (Vite, Electron, React, Backend scripts).
    - **User Guide**: Installation, importing data (Nubank, Binance, etc.), and basic usage.
    - **Dev Guide**: Setting up the development environment, running tests, and contribution guidelines.
- Ensure the site is responsive and visually consistent with the MondayMoney brand.

## Goals
- Improve user onboarding.
- Provide a clear reference for developers.
- Centralize project information currently scattered across READMEs and docs folders.
